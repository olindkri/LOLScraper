# Spec B: LP Gain/Loss Display

## Goal

Scrape LP gain/loss for each newly fetched ranked game and display it inline with the Result column in the expanded game table, so players can see how much LP they won or lost in each game.

## Architecture

Two independent changes sharing the same deploy:
1. **Backend** ‚Äî extend `scraper/scrape.py` to scrape `lpDelta` per game row.
2. **Frontend** ‚Äî update `frontend/src/components/ScoreboardRow.jsx` to render LP inline with the result.

Old cached games in Firebase are missing `lpDelta` ‚Äî frontend treats both absent and `null` as "no LP to show", displaying only WIN/LOSS as before.

## Tech Stack

Python 3 (scraper), React + inline styles (frontend), Firebase Realtime DB, JetBrains Mono font, existing CSS variable design system.

---

## Section 1: Scraper Update

### New Field

**`lpDelta: int | None`** ‚Äî added to each game dict in `_parse_rows`. Positive for gains (e.g. `+18 LP` ‚Üí `18`), negative for losses (e.g. `-12 LP` ‚Üí `-12`), `None` when LP text is absent (game too fresh, or not calculated yet).

### Scraping

LP change is displayed in the `resultCellDark` td on leagueofgraphs, typically as text matching `[+-]\d+ LP`. Parsed with:

```python
lp_delta = None
lp_text = dark_td.get_text()
lp_match = re.search(r'([+-]\d+)\s*LP', lp_text)
if lp_match:
    lp_delta = int(lp_match.group(1))
```

Added to the game dict alongside `result`, `champion`, etc.

### Graceful Degradation

Games already in Firebase have no `lpDelta` key. The frontend handles `null` and absent identically ‚Äî shows nothing.

### Files

- Modify: `scraper/scrape.py` ‚Äî add `lpDelta` extraction in `_parse_rows`
- Modify: `scraper/tests/test_scrape.py` ‚Äî add tests for new field

## Section 2: Frontend Display

### Result Cell Change

In `ScoreboardRow.jsx`, the Result cell currently renders `WIN` or `LOSS` with a colored square. When `lpDelta` is present (not null/undefined):

- **WIN +18**: `WIN` in `--win`, then ` +18` bold in `--win`
- **LOSS -12**: `LOSS` in `--loss`, then ` -12` bold in `--loss`
- Positive `lpDelta` always rendered with explicit `+` prefix
- When `lpDelta` is null/undefined: unchanged ‚Äî `WIN` or `LOSS` only

All text stays in `--font-mono`. The LP delta is a `<span>` with `fontWeight: 700` immediately after the result text, same color as the result.

### Files

- Modify: `frontend/src/components/ScoreboardRow.jsx` ‚Äî add LP delta rendering in Result cell
